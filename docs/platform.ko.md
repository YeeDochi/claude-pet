# 플랫폼 지원

[← README](../README.ko.md) · [English](platform.md) | **한국어**

반응 코어(상태·애니메이션·배회·드래그/던지기·트레이)는 이식 가능해요. 창 통합(올라타기/가림·
클릭-포커스)은 Windows에서도 폴링 기반 Win32 API로 동작하고, macOS용 폴링 기반 Quartz
구현도 있지만 **실기 테스트를 전혀 못 한 추측성 코드**예요 — 아래 참고. KDE 전용으로 남은 건
작업표시줄 숨김뿐이에요. 그 외 미구현 기능은 그냥 꺼져요 — 펫은 여전히 실행돼요.

| 플랫폼 | 실행 | 창 통합 |
|--------|------|---------|
| **KDE Plasma** (Wayland/X11) | ✅ | ✅ 전부 — 올라타기, 가림 클리핑/숨김, 클릭-포커스, 작업표시줄 숨김 |
| 기타 Linux (GNOME 등) | ✅ (XWayland) | ✖ KDE 전용 부분 no-op (배회/드래그/상태/트레이는 동작) |
| **Windows** | ✅ | ✅ 올라타기, 가림 클리핑/숨김, 클릭-포커스(`SetForegroundWindow`, 폴링 기반 `ctypes`/Win32); ✖ 작업표시줄 숨김 미구현 |
| **macOS** | 🅱️ 실행될 것 (네이티브 Qt) | 🧪 올라타기/가림 구현됨(`pyobjc` 폴링 기반 Quartz) — 단 **미검증: Mac 없이 작성, 실기 테스트 필요**; ⚠️ `osascript` 기반 클릭-포커스(best-effort) |

CLI 도구(`bin/*`)는 전부 **Python** — bash 없음 — 이라 Python 되는 곳이면 어디서든 실행돼요.
KDE와 Windows는 실제로 테스트했어요. GNOME은 창 통합 범위 밖이에요. macOS 창 통합 코드는
**Mac 실기가 전혀 없는 상태에서 문서만 보고 작성**했어요 — 동작하는 기능이 아니라 테스트용
출발점으로 봐주세요. 창 통합이 미구현인 곳(또는 macOS에서 pyobjc가 없는 경우)에선 바탕화면
바닥으로 폴백하고 해당 기능은 꺼져요.

## 요구사항

- Python 3 + PyQt6 — `pip install PyQt6`
- 전체 기능엔 **KDE Plasma**: `qdbus6`(창 통합/클릭-포커스), `wmctrl`(선택, 작업표시줄에서
  펫 숨김). Wayland면 XWayland.
- **Windows**: 추가 요구사항 없음 — 창 통합은 `src/windows_win32.py`에서 표준 라이브러리
  `ctypes`로 `user32`/`dwmapi`/`kernel32`를 직접 호출해요.
- **macOS 전용**: 올라타기/가림에 `pip install pyobjc-framework-Quartz`
  (`src/macos_geom.py`). 선택 사항 — 없어도 펫은 실행되고, 창 통합만 꺼져요.
  Windows/Linux에선 절대 필요 없고 import되지도 않아요.

## 각자 OS에서 테스트 도와주세요

macOS 창 통합 경로 전체(`src/macos_geom.py`와 `pet.py`의 연결부)는 **Mac 실기 없이
Apple 문서만 보고 작성한 추측성 코드**로, 한 번도 macOS에서 실행된 적이 없어요.
실제로 테스트해 줄 사람에게 넘기려고 만든 거예요 — 리포트 정말 환영해요. 돌려보면
대략 이 순서로 확인 후 이슈로:

1. **macOS 첫 단계 — 화면 기록(Screen Recording) 권한.** 시스템 설정 → 개인정보 보호
   및 보안 → 화면 기록에서 펫을 실행하는 터미널(또는 Python)에 권한을 주세요. 없으면
   macOS가 창 *제목*을 열거 API에서 숨겨요. 코드는 권한이 필요 없는 앱 이름으로
   폴백하니 올라타기/가림은 권한 없이도 될 *것*으로 예상해요 — 권한 있는/없는 상태
   둘 다 테스트해서 차이를 알려주세요.
2. **실행되나요?** `bin/claude-pet`로 크리처가 뜨고, 배회하고, 드래그/던지기가 되나요.
   (창 통합용으로 먼저 `pip install pyobjc-framework-Quartz`.)
3. **올라타나요?** 펫을 다른 창 위로 드래그하면 발이 그 창의 상단 모서리에 얹히고,
   창을 옮기면 따라가나요. 높이가 뒤집혀/어긋나게 얹히면 `src/macos_geom.py`의 좌표계
   가정(파일에 문서화됨)이 틀린 거예요. 보조 모니터도 확인해 주세요.
4. **가림 처리되나요?** 펫이 얹힌 창 위로 다른 창을 올리면 펫이 잘리거나 숨나요.
   *반대로* 동작하면 `src/macos_geom.py`의 z-순서 가정이 틀린 거예요.
5. **반응하나요?** `claude-pet-install` 후 Claude Code를 쓰면 상태가 바뀌나요 (작업/생각/완료).
6. **트레이** 아이콘과 메뉴가 뜨나요.
7. **클릭-포커스:** 좌클릭이 터미널/IDE(Terminal / iTerm / VS Code)를 앞으로 가져오나요
   (`osascript`). 최전면 앱 감지가 "완료" 포즈를 게이팅하나요.
- 위 표 대비 뭐가 안 되는지 적어주세요 (작업표시줄 숨김은 설계상 KDE 전용이고,
  세션별 호스트 창 추적 — 호스트 최소화 시 숨김 — 은 macOS 미구현이에요).
